# MATLAB VNF 部署仿真批量实验平台使用说明

## 一、项目概述

本项目是一个 **VNF（虚拟网络功能）/ SFC（服务功能链）部署仿真实验平台**。当前推荐入口是 `run_batch_experiments.m`，通过 `a.输入/批量实验配置.xlsx` 驱动批量实验，自动完成拓扑与请求准备、部署算法执行、结果汇总、绘图数据聚合和 SVG 图表生成。

核心能力：
- Excel 配置驱动：拓扑、请求、方法、参数组、全局绘图设置都来自 `批量实验配置.xlsx`
- 支持多拓扑、多算法、多参数组、多请求集、多次重复
- 支持断点续跑：已成功完成的 `case_id` 会自动跳过
- 自动输出 `实验结果汇总.xlsx`、`绘图数据.xlsx` 和图表文件
- 在线 RDA 支持消融实验开关：共享潜力权重衰减、动态目的节点接入

## 三、快速开始

1. 在 MATLAB 中把当前目录切到项目根目录。
2. 运行：

```matlab
run_batch_experiments
```

3. 查看输出：
- `c.输出/实验结果汇总.xlsx`：每个实验 case 的原始结果
- `c.输出/绘图数据.xlsx`：按方法/参数组聚合后的绘图数据
- `c.输出/5.结果图保存/`：自动生成的 SVG 图

如需重跑全部实验，删除 `c.输出/实验结果汇总.xlsx` 后重新运行。只想重跑部分实验时，删除汇总表里对应 `case_id` 的记录，再运行主入口。

## 四、配置文件说明

`a.输入/批量实验配置.xlsx` 是当前唯一推荐配置入口，包含以下 sheet：

| Sheet | 作用 |
|---|---|
| `实验组合` | 定义要运行的拓扑、方法、参数组、请求集和重复次数 |
| `拓扑配置` | 定义拓扑函数和节点/链路资源范围 |
| `请求配置` | 定义请求数量、目的节点数、VNF 数、资源需求和时延约束 |
| `方法配置` | 注册部署算法函数、显示名、请求类型和在线/离线模式 |
| `方法参数组` | 定义算法参数和 RDA 消融开关 |
| `全局设置` | 定义输出路径、绘图尺寸、字体、SVG 设置等 |

### 实验组合

常用字段：
- `group_id`：实验组名
- `topo_name`：拓扑名，对应 `拓扑配置`
- `method_name`：方法名，对应 `方法配置`
- `param_group`：参数组名，对应 `方法参数组`
- `request_set_ids`：请求集编号，支持 `1` 或 `1,2,3`
- `repeat_count`：重复次数
- `enabled`：`1` 启用，`0` 跳过

### 方法配置

当前推荐方法：
- `RDA`：在线资源与时延感知算法，实际函数为 `ResourceAndDelayAwareOnline`
- `nodeFirst`：节点资源贪心
- `shortestPathFirstWithLoadBalancing`：最短路优先/负载均衡
- `RSA`：随机部署

离线 `ResourceAndDelayAware` 已弃用并删除；RDA 相关新实验请统一使用方法名 `RDA`。

### RDA 消融参数组

`方法参数组` 中 RDA 支持以下参数：

| 字段 | 说明 |
|---|---|
| `candLinkNum` | 候选链路/路径数量 |
| `candNodeNum` | 候选节点数量 |
| `shareWeight` | 共享潜力权重 |
| `congWeight` | 拥塞/资源权重 |
| `delayWeight` | 时延权重 |
| `shareDecayMin` | 共享潜力权重衰减下限 |
| `enableShareDecay` | 是否启用共享潜力权重衰减，默认启用 |
| `enableDynamicDestAccess` | 是否启用动态目的节点接入，默认启用 |

布尔字段可写 `1/0`、`true/false`、`on/off`。旧配置没有这两列时，默认都视为启用。

当前内置 2x2 消融组：

| param_group | enableShareDecay | enableDynamicDestAccess | 含义 |
|---|---:|---:|---|
| `baseline` | 1 | 1 | 两个机制都启用 |
| `noShareDecay` | 0 | 1 | 只关闭共享潜力权重衰减 |
| `noDynamicDest` | 1 | 0 | 只关闭动态目的节点接入 |
| `bothOff` | 0 | 0 | 两个机制都关闭 |

关闭 `enableShareDecay` 只会把共享潜力衰减因子固定为 1，不会关闭共享潜力评分，也不会关闭实际 VNF/链路共享。

## 五、执行流程

```text
批量实验配置.xlsx
  -> load_config
  -> build_experiment_cases
  -> run_single_experiment
       -> prepare_topo
       -> prepare_requests
       -> execute_deploy_pipeline
       -> collect_experiment_metrics
  -> write_experiment_result
  -> compute_plot_data
  -> plot_from_xlsx
```

`execute_deploy_pipeline` 根据 `online_mode` 调用在线或离线流程。当前 RDA 使用在线模式：逐请求生成候选多播方案、在线评估排序、逐个尝试部署，成功则提交资源状态，失败则回滚并尝试备选方案。

## 六、绘图与图例

`compute_plot_data` 会在绘图数据中生成：

```text
legend_name = display_name + "-" + param_group
```

因此 RDA 消融图例会显示为：
- `RDA-baseline`
- `RDA-noShareDecay`
- `RDA-noDynamicDest`
- `RDA-bothOff`

各绘图函数优先使用 `legend_name`；如果读取的是旧版 `绘图数据.xlsx`，没有 `legend_name` 时会自动退回使用 `display_name`。

## 七、扩展指南

### 添加新拓扑

1. 在 `a.输入/1.原始拓补信息/` 新增拓扑函数，返回邻接矩阵。
2. 在 `拓扑配置` 增加一行。
3. 在 `请求配置` 增加对应请求生成参数。
4. 在 `实验组合` 增加实验行。

### 添加新算法

1. 在 `b.常用函数/3.部署方案生成/` 增加算法函数。
2. 在 `方法配置` 注册 `method_name`、`display_name`、`deploy_func`、`online_mode` 等字段。
3. 如有算法参数，在 `方法参数组` 增加参数列和参数组。
4. 在 `实验组合` 中引用新方法和参数组。

在线算法推荐签名：

```matlab
[plan, nodes, links, consume, fail_log, deployStats] = your_algorithm( ...
    requests, KPathsNew, links, nodes, plan, consume, fail_log, deploy_cfg)
```

### 添加新指标或图表

1. 在 `collect_experiment_metrics.m` 增加指标提取。
2. 在 `write_experiment_result.m` 写入汇总表。
3. 在 `compute_plot_data.m` 增加聚合逻辑。
4. 在 `b.常用函数/6.结果绘制/2.绘图函数/` 增加绘图函数。
5. 在 `plot_from_xlsx.m` 调用新绘图函数。

## 八、常见问题

### 配置文件不存在

首次运行时会自动调用 `create_default_config_xlsx` 生成默认模板。

### 图表里同一方法多条线如何区分

使用 `param_group` 区分。同一 `display_name` 的不同参数组会自动生成不同 `legend_name`。

### 中文显示乱码

MATLAB / PowerShell 读取中文文件时建议显式指定 UTF-8。PowerShell 示例：

```powershell
Get-Content -Encoding UTF8 -Raw 'z.说明文件\使用说明.md'
```

### KPaths 计算慢

KPaths 首次生成后会缓存到 `c.输出/1.拓补信息/` 和 `a.输入/2.整理后的拓补信息/`，后续运行会优先加载缓存。
