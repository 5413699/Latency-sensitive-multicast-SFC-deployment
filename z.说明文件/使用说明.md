# MATLAB VNF 部署仿真 — 批量实验平台使用说明

## 一、项目概述

本项目是一个 **VNF（虚拟网络功能）部署仿真实验平台**，支持在多种网络拓扑上，使用不同的部署算法，对 SFC（服务功能链）请求进行批量仿真实验，并自动汇总结果、生成对比图表。

核心能力：
- 通过 Excel 配置文件驱动批量实验（多拓扑 x 多方法 x 多参数 x 多请求集 x 多次重复）
- 实验结果统一写入 Excel 汇总表
- 自动聚合多次重复实验的均值/标准差/置信区间
- 一键生成全部对比图表（SVG 格式）
- 支持断点续跑，中断后可从上次进度恢复

## 三、快速开始

### 第 1 步：打开 MATLAB 并设置工作目录

在 MATLAB 中将当前目录切换到项目根目录 `02-实验代码/`。

### 第 2 步：运行主入口脚本

```matlab
run_batch_experiments
```

首次运行时，如果 `a.输入/批量实验配置.xlsx` 不存在，系统会自动生成包含默认配置的模板文件。

### 第 3 步：查看结果

实验完成后，可在以下位置查看结果：
- `c.输出/实验结果汇总.xlsx` — 全部实验的原始指标数据
- `c.输出/绘图数据.xlsx` — 聚合统计后的绘图数据
- `c.输出/5.结果图保存/` — SVG 图表文件

## 四、配置文件详解

配置文件 `a.输入/批量实验配置.xlsx` 包含 **6 个 Sheet**，是整个实验平台的唯一配置源。

### Sheet 1：实验组合

定义要运行哪些实验组合。每行代表一组实验。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `group_id` | string | 实验组标识，用于分组管理 | `G1` |
| `topo_name` | string | 拓扑名称，对应"拓扑配置"sheet 中的行 | `US_Backbone` |
| `method_name` | string | 方法名称，对应"方法配置"sheet 中的行 | `ResourceAndDelayAwareOnline` |
| `param_group` | string | 参数组名称，对应"方法参数组"sheet 中的行 | `default` |
| `request_set_ids` | string | 请求集编号，逗号分隔支持多集 | `1` 或 `1,2,3` |
| `repeat_count` | number | 每个组合的重复次数 | `3` |
| `enabled` | number | 1=启用, 0=跳过 | `1` |

**关键说明：**
- 每行会与 `request_set_ids` 和 `repeat_count` 交叉展开。例如 `request_set_ids=1,2` + `repeat_count=3` 会展开为 6 次实验
- `enabled=0` 的行会被跳过，便于临时禁用某些组合

### Sheet 2：拓扑配置

定义网络拓扑的资源参数。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `topo_name` | string | 拓扑标识（与实验组合对应） | `US_Backbone` |
| `topo_func` | string | 拓扑生成函数名（需在 MATLAB 路径上） | `US_Backbone` |
| `minm` / `maxm` | number | 节点内存资源范围 | `80` / `120` |
| `minc` / `maxc` | number | 节点 CPU 资源范围 | `80` / `120` |
| `minb` / `maxb` | number | 链路带宽资源范围 | `160` / `320` |

### Sheet 3：请求配置

定义 SFC 请求的生成参数。按 `topo_name` 关联拓扑。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `topo_name` | string | 所属拓扑 | `US_Backbone` |
| `requests_num` | number | 请求总数 | `100` |
| `destNode_count` | number | 每请求最大目的节点数 | `5` |
| `vnftype_num` | number | VNF 类型总数 | `8` |
| `vnf_num` | number | 每请求 VNF 数量 | `3` |
| `maxbw` / `minbw` | number | 带宽需求范围 | `8` / `2` |
| `maxnr` / `minnr` | number | 资源需求范围 | `4` / `1` |
| `maxt` / `mint` | number | 时延约束范围 | `250` / `100` |

### Sheet 4：方法配置

注册可用的部署算法。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `method_name` | string | 方法内部标识 | `ResourceAndDelayAwareOnline` |
| `display_name` | string | 图表显示名称（简短） | `RDA` |
| `deploy_func` | string | 部署算法函数名 | `ResourceAndDelayAwareOnline` |
| `requests_type` | string | 使用的请求集类型 | `sortedRequests` 或 `requests` |
| `fixed_func` | string | 固定树计划函数 | `FixedTreePlan` |
| `sorted_func` | string | 排序计划函数 | `generateDeployPlan` |
| `online_mode` | number | 0=离线模式, 1=在线模式 | `1` |

**在线/离线模式的区别：**
- **离线模式** (`online_mode=0`)：先生成全局部署方案，再统一映射。适用于 STB、NIF、RSA 等算法
- **在线模式** (`online_mode=1`)：逐请求即时部署。适用于 RDA 等在线算法

### Sheet 5：方法参数组

为特定方法定义参数变体，用于参数调优对比实验。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `method_name` | string | 关联的方法名 | `ResourceAndDelayAwareOnline` |
| `param_group` | string | 参数组标识 | `default` |
| `candLinkNum` | number | 候选链路数 | `5` |
| `candNodeNum` | number | 候选节点数 | `5` |
| `shareWeight` | number | 共享权重 | `1` |
| `congWeight` | number | 拥塞权重 | `1.0` |
| `delayWeight` | number | 时延权重 | `3.0` |
| `shareDecayMin` | number | 共享衰减下限 | `0` |

**参数组用法：**
- 同一方法可定义多个参数组（如 `default`、`aggressive`、`conservative`）
- 在"实验组合"sheet 中通过 `param_group` 字段指定使用哪个参数组
- 不在"方法参数组"中的方法默认使用基础参数

### Sheet 6：全局设置

key-value 格式的全局参数。

| key | 默认值 | 说明 |
|-----|--------|------|
| `rng_seed_base` | `42` | 随机种子基数，保证实验可重复 |
| `output_dir` | `c.输出` | 输出根目录 |
| `summary_xlsx` | `实验结果汇总.xlsx` | 结果汇总文件名 |
| `plot_data_xlsx` | `绘图数据.xlsx` | 绘图数据文件名 |
| `fig_visible` | `off` | 绘图时是否显示窗口（`on`/`off`） |
| `save_svg` | `1` | 是否保存 SVG 图表（1=是, 0=否） |
| `save_mat` | `0` | 是否保存 MAT 结果文件 |
| `svg_background` | `none` | SVG 背景色 |
| `fig_width` | `800` | 默认图宽（像素） |
| `fig_height` | `600` | 默认图高（像素） |
| `line_width` | `1.8` | 曲线线宽 |
| `font_size` | `12` | 图表字号 |
| `slack_mode` | `ratio` | 裕量计算模式（`ratio`=比例, `abs`=绝对值） |
| `e2e_source` | `auto` | 端到端时延来源 |

## 七、绘图函数使用

所有绘图函数均支持**双模式**调用：

### 模式 1：批量调用（由 plot_from_xlsx 驱动）

```matlab
% 一键生成全部图表

% 或指定数据文件和配置

```

### 模式 2：独立运行（单独调试某个图）

```matlab
% 无参调用 — 自动从默认路径加载 xlsx 数据和全局配置
plot_acceptance_rate_bar()
plot_blocking_rate_curve()
plot_cumulative_resource()
plot_failure_distribution()
plot_vnf_sharing_gain()

% 平均指标曲线（通过参数指定指标类型）
plot_avg_metric_vs_success()                     % 默认绘制 avg_cpu
plot_avg_metric_vs_success([], 'avg_mem')         % 绘制 avg_mem
```

## 八、断点续跑

`run_batch_experiments` 内置了断点续跑机制：

1. 每次实验完成后，结果会立即追加写入 `实验结果汇总.xlsx`
2. 重新运行脚本时，系统会读取已有的汇总表，识别状态为 `success` 的 case_id
3. 已完成的实验会被自动跳过，只运行剩余实验

**使用方式：** 中断后直接再次运行 `run_batch_experiments` 即可，无需任何额外操作。

**清空重跑：** 如需从头开始，删除 `c.输出/实验结果汇总.xlsx` 后重新运行。

## 九、扩展指南

### 添加新的部署算法

1. 在 `b.常用函数/3.部署方案生成/` 下创建新算法目录和函数
2. 在 `批量实验配置.xlsx` 的"方法配置"sheet 中新增一行，填写函数名和模式
3. 在"实验组合"sheet 中添加使用该方法的实验行

**在线算法函数签名：**
```matlab
[plan, nodes, links, consume, fail_log, ~] = your_algorithm( ...
    requests, KPathsNew, links, nodes, plan, consume, fail_log, deploy_cfg)
```

**离线算法函数签名：**
```matlab
plan = your_algorithm(requests, KPathsNew, links, nodes, plan, deploy_cfg)
```

### 添加新的拓扑

1. 在 `a.输入/1.原始拓补信息/` 中创建拓扑函数（如 `MyTopo.m`），返回邻接矩阵
2. 在"拓扑配置"sheet 中新增一行
3. 在"请求配置"sheet 中为新拓扑添加请求生成参数
4. 在"实验组合"sheet 中添加使用该拓扑的实验行

### 添加新的指标和图表

1. 在 `collect_experiment_metrics.m` 中新增指标提取逻辑
2. 在 `write_experiment_result.m` 中将新指标写入汇总表
3. 在 `compute_plot_data.m` 中新增聚合逻辑
4. 在 `b.常用函数/6.结果绘制/2.绘图函数/` 中创建新的绘图函数
5. 在 `plot_from_xlsx.m` 中添加调用

### 强化学习（RL）方法扩展

`execute_deploy_pipeline.m` 已预留 RL 扩展接口：

```matlab
% 在 execute_deploy_pipeline.m 的条件分支中:
% elseif isfield(method_cfg, 'rl_mode') && method_cfg.rl_mode
%     [plan, nodes, links, consume, fail_log] = rl_deploy(...)
```

接入步骤：
1. 实现 RL 部署函数
2. 取消上述注释并适配接口
3. 在"方法配置"sheet 中新增 RL 方法行，添加 `rl_mode` 参数

## 十、常见问题排查

### Q: 提示"配置文件不存在"
**A:** 正常现象。首次运行时系统会自动调用 `create_default_config_xlsx` 生成默认配置。生成后可按需修改。

### Q: 实验运行报错但不影响整体
**A:** 单个实验的错误会被 try-catch 捕获，状态记为 `error` 并写入汇总表，不影响后续实验执行。

### Q: 想重跑部分实验
**A:** 在 `实验结果汇总.xlsx` 的"实验元数据"sheet 中删除对应行（注意同步删除"汇总指标"和"请求级明细"中的对应数据），然后重新运行。

### Q: 图表中文显示异常
**A:** 检查 MATLAB 的字体设置，确保系统安装了中文字体。可在全局设置中将 `fig_visible` 设为 `on` 以便实时查看图表效果。

### Q: Excel 读写速度较慢
**A:** 这是 MATLAB Excel I/O 的固有限制。对于大规模实验（>100 个 case），建议：
- 将 `repeat_count` 控制在合理范围（3-5 次）
- 分批运行不同的实验组（通过 `enabled` 字段控制）

### Q: KPaths 计算耗时很长
**A:** KPaths 计算只在首次运行时发生，结果会缓存到 `c.输出/1.拓补信息/` 和 `a.输入/2.整理后的拓补信息/`。后续运行会自动加载缓存。
