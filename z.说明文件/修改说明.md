# 项目修改说明

## 2026-05-12：RDA 消融实验配置化与结构整理

### 一、修改目标

本次修改围绕在线 RDA 消融实验展开：
- 通过 `批量实验配置.xlsx` 控制 RDA 的两个机制开关
- 为完整 2x2 消融实验提供默认配置
- 让绘图结果能区分同一方法的不同参数组
- 删除已弃用的离线 RDA 旧实现
- 更新说明文件，使项目入口和目录职责更清晰

### 二、RDA 新增配置开关

在 `方法参数组` sheet 中新增两个字段：

| 字段 | 默认值 | 说明 |
|---|---:|---|
| `enableShareDecay` | 1 | 是否启用共享潜力权重衰减 |
| `enableDynamicDestAccess` | 1 | 是否启用动态目的节点接入 |

布尔值支持 `1/0`、`true/false`、`on/off`。旧配置没有这两列时，代码按默认启用处理。

### 三、2x2 消融参数组

当前 `批量实验配置.xlsx` 和默认模板均包含以下 RDA 参数组：

| param_group | enableShareDecay | enableDynamicDestAccess |
|---|---:|---:|
| `baseline` | 1 | 1 |
| `noShareDecay` | 0 | 1 |
| `noDynamicDest` | 1 | 0 |
| `bothOff` | 0 | 0 |

同时在 `实验组合` 中新增 `rdaAblation` 实验组，用于直接运行上述 4 组消融对比。

### 四、代码改动

| 文件 | 修改内容 |
|---|---|
| `ResourceAndDelayAwareOnline.m` | 新增 `enableDynamicDestAccess` 开关；关闭时按原始目的节点顺序接入 |
| `calcPathScore.m` | 新增 `enableShareDecay` 开关；关闭时共享衰减因子固定为 1 |
| `calcNodeScore.m` | 新增 `enableShareDecay` 开关；关闭时共享衰减因子固定为 1 |
| `compute_plot_data.m` | 输出 `legend_name = display_name + "-" + param_group` |
| `plot_*.m` | 绘图函数优先使用 `legend_name`，兼容旧数据的 `display_name` |
| `create_default_config_xlsx.m` | 默认模板新增 RDA 消融开关列和 2x2 消融参数组 |

### 五、删除弃用文件

离线 RDA 已弃用，已删除：

| 删除文件 | 原因 |
|---|---|
| `b.常用函数/3.部署方案生成/9.资源与时延感知算法/ResourceAndDelayAware.m` | 离线 RDA 旧入口已不再使用 |
| `b.常用函数/3.部署方案生成/9.资源与时延感知算法/5.源到单个目的节点函数/rdaLinkAndNodeChoice.m` | 仅服务离线 RDA 的旧单目的节点选择逻辑 |

后续 RDA 实验请统一使用 `方法配置` 中的 `RDA`，其 `deploy_func` 为 `ResourceAndDelayAwareOnline`。

### 六、项目结构整理原则

本次整理采用“低风险整理”：
- 保留现有中文目录名和脚本查找路径，避免破坏 MATLAB `addpath(genpath(...))` 和历史输出路径
- 明确推荐入口：`run_batch_experiments.m`
- 明确推荐配置源：`a.输入/批量实验配置.xlsx`
- 明确当前 RDA 只保留在线实现
- 文档中重新梳理了输入、函数库、输出、论文绘图、历史归档的职责

暂未做大规模目录搬迁；如后续需要把历史输出、论文图、核心代码进一步拆成更干净的工程结构，建议单独开一轮迁移，并同步修正所有路径引用和历史脚本。
