# 批量实验平台重构 — 修改说明

## 一、重构目标

将原有"手动逐脚本运行单次实验"的架构，重构为"Excel 配置驱动、批量实验自动化、结果统一汇总、绘图解耦"的实验平台架构。

核心改动原则：
- **不保留任何旧入口和兼容层**，直接采用全新架构
- **不修改核心算法逻辑**，仅重构执行流程和数据流
- **xlsx 驱动一切**：配置、结果汇总、绘图数据全部通过 Excel 文件流转

---

## 二、新增文件清单

### 2.1 主入口

| 文件 | 职责 |
|------|------|
| `run_batch_experiments.m` | 批量实验主入口：读配置 → 展开组合 → 逐个执行 → 写结果 → 计算绘图数据 → 绘图 |

### 2.2 批量实验框架（`b.常用函数/7.批量实验框架/`）

| 文件 | 职责 |
|------|------|
| `create_default_config_xlsx.m` | 生成 `批量实验配置.xlsx` 默认模板（6个sheet） |
| `load_config.m` | 读取 xlsx 所有 sheet 到 cfg struct |
| `read_global_cfg.m` | 读取全局绘图参数，供绘图函数独立运行时使用 |
| `build_experiment_cases.m` | 将 cfg 展开为 cases 数组（每个元素=一次实验） |
| `prepare_topo.m` | 拓扑准备（复用旧拓扑生成逻辑，含缓存） |
| `prepare_requests.m` | 请求生成/加载（复用旧请求生成逻辑，含种子控制） |
| `execute_deploy_pipeline.m` | 部署执行（复用旧部署+映射逻辑，预留RL扩展点） |
| `run_single_experiment.m` | 单次实验编排：prepare_topo → prepare_requests → execute_deploy → collect_metrics |
| `collect_experiment_metrics.m` | 指标采集：从 requests/nodes/consume/fail_log 提取汇总和逐请求指标 |
| `write_experiment_result.m` | 将单次实验结果追加写入 `实验结果汇总.xlsx` |
| `compute_plot_data.m` | 从 `实验结果汇总.xlsx` 聚合统计，输出 `绘图数据.xlsx` |
| `plot_from_xlsx.m` | 主绘图入口：从 `绘图数据.xlsx` 一键调用全部绘图函数 |

### 2.3 新绘图函数（`b.常用函数/6.结果绘制/2.绘图函数/`）

所有绘图函数采用**双模式设计**：
- **批量模式**：由 `plot_from_xlsx.m` 传入数据表和配置
- **独立模式**：无参调用时自动从 xlsx 加载数据 + `read_global_cfg()` 获取配置

| 文件 | 数据源 sheet | 对应旧函数 |
|------|-------------|-----------|
| `plot_acceptance_rate_bar.m` | 方法对比汇总 | （新增） |
| `plot_avg_metric_vs_success.m` | 逐请求曲线 | plot_avg_cpu/mem/bw/delay/slack_vs_success（5合1模板） |
| `plot_blocking_rate_curve.m` | 逐请求曲线 | plot_blocking_rate_curve |
| `plot_cumulative_resource.m` | 逐请求曲线 | plot_cumulative_resource_consume |
| `plot_failure_distribution.m` | 失败分布 | plot_failure_breakdown_distribution |
| `plot_vnf_sharing_gain.m` | 逐请求曲线 | plot_vnf_sharing_gain_ratio_vs_success |
| `plot_gantt_chart.m` | 运行时 nodes struct | plot_gantt_chart |

## 五、数据流设计

```

        │
        ▼
  ┌─ load_config ─┐
  │               │
  ▼               ▼
build_experiment_cases
        │
        ▼ (cases 数组)
  ╔═══════════════╗
  ║ 逐个执行循环   ║
  ║               ║
  ║ prepare_topo  ║
  ║      ↓        ║
  ║ prepare_reqs  ║
  ║      ↓        ║
  ║ execute_pipeline ║
  ║      ↓        ║
  ║ collect_metrics ║
  ║      ↓        ║
  ║ write_result  ║
  ╚═══════════════╝
        │
        ▼
  实验结果汇总.xlsx (3 sheets)
        │
        ▼
  compute_plot_data
        │
        ▼
  绘图数据.xlsx (3 sheets)
        │
        ▼
  plot_from_xlsx → 7 个绘图函数
        │
        ▼
  SVG 图表输出
```

## 六、xlsx 数据模型

### 6.1 批量实验配置.xlsx（输入）

| Sheet | 说明 |
|-------|------|
| 实验组合 | 每行=一组实验（topo_id, req_set_ids, method_id, param_group_id, repeats） |
| 拓扑配置 | 拓扑参数（topo_id, topo_func, node_count...） |
| 请求配置 | 请求参数（req_set_id, request_num, vnf_types, seed...） |
| 方法配置 | 方法注册（method_id, display_name, deploy_func, online_mode...） |
| 方法参数组 | 方法特有参数（param_group_id, method_id, 参数键值对...） |
| 全局设置 | key-value 全局参数（output_dir, fig_visible, line_width...） |

### 6.2 实验结果汇总.xlsx（中间产物）

| Sheet | 说明 |
|-------|------|
| 实验元数据 | case_id, topo_name, display_name, param_group, repeat_id, seed, status, elapsed_s |
| 汇总指标 | case_id, acceptance_rate, avg_cpu, avg_memory, avg_bandwidth, avg_e2e_delay, avg_slack_ratio, total_*, vnf_sharing_gain, fail_* |
| 请求级明细 | case_id, req_id, deploy_order, accepted, cpu/mem/bw_consume, e2e_delay, max_delay, slack_ratio, dest_count, vnf_count, vnf_gain |

### 6.3 绘图数据.xlsx（绘图输入）

| Sheet | 说明 |
|-------|------|
| 方法对比汇总 | topo_name, display_name, param_group, metric_name, mean, std, ci95_lo, ci95_hi, n |
| 逐请求曲线 | topo_name, display_name, param_group, success_index, avg_cpu_mean/std, avg_mem_mean/std, ... |
| 失败分布 | topo_name, display_name, param_group, fail_type, count_mean, count_std |

## 七、关键设计决策

### 7.1 双模式绘图

每个绘图函数支持两种调用方式：
```matlab
% 批量模式（由 plot_from_xlsx 调用）
plot_acceptance_rate_bar(compare_tbl, outDir, cfg)

% 独立模式（直接运行调试）
plot_acceptance_rate_bar()  % 自动加载 xlsx + read_global_cfg
```

### 7.2 指标函数 5 合 1

旧架构中 CPU/内存/带宽/时延/裕量 各有独立的绘图函数，新架构合并为 `plot_avg_metric_vs_success.m` 统一模板，通过 `metric_field` 参数区分。

### 7.3 RL 扩展点

`execute_deploy_pipeline.m` 中预留了 RL 方法集成接口：
```matlab
if deploy_cfg.rl_mode
    % 未来 RL 方法在此接入
end
```

### 7.4 断点续跑

`run_batch_experiments.m` 支持 checkpoint 机制，通过 `checkpoint.mat` 记录已完成的 case_id，重启后自动跳过已完成实验。
